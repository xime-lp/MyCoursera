The Box Model and Layout

The Box Model

In HTML every element is considered a box and has certain elements shown in this diagram.

Definitions

  • The content is whatever is inside the element. If we have a div with words in it, the content would be all the space the words took. Height and width properties affect this part of the element.(orange
  • The padding is the space between the content and the border, it is a part of the element itself. For example if we have a div with a height of 50px and we add 10px of padding to the top and bottom, the div would occupy a total of 70px. (salmon)
  • The border is the outmost layer of the box. It is a part of the element itself and, like the padding, affects sizing. (black)
  • The margin is the space between the element and any other element. It does not affect the size of the element and it's not part of it. (white) More later.
  • We can resize the padding and the margins like so – margin/padding: (top)0 (right)0 (bottom)0 (left)0;
  • The border is set like so – border: (weight)0px (style)solid (color)black
This is the content

Sizing a box

Every HTML element has a property called box-sizing and by default it is set to content. We can change this to avoid sizing problems when it comes to padding and borders. In order to include these two properties in our sizing we can set the box-sizing to border-box in CSS.

Unfortunately, box-sizing is not inheritable like other selectors. But we can use the "*" selector that says that those properties should be applied to everything, it's not inherited.

Margins

  • They are horizontally comulative.
  • However, margins vertically collapse. The larger margin will win.
  • To override default browser margins, we can put the commands in the "*" selector.

Constraining size

When we constrain an element to a certain size, content might overflow and we have a way to solve it. We use the command overflow: value; And we have some options:

  • visible – the overflowing content will be visible.
  • hidden – it clips the content where the element ends
  • auto – adds a scroller
  • scroll – adds a scroller on both sides even if they're not used.

The background Property

Backgrounds have various properties such as:

  • background-color: (color) – sets a color for the background
  • background-image: url("file") – sets an image as the background. It wins in priority vs the color
  • background-repeat: (repeat, no-repeat, repeat-x, repeat-y) – it tells how many times to repeat the background image over the element.
  • background-position: (vertical) (horizontal) – where the background image will be positioned.
  • background: (image) (repeat) (position) (color) – sets all the properties at once. It overrides other background properties.

Positioning elements by floating

When you float elements, you take them out of the document flow. When it comes to floated elements, the margins never collapse. Floating is alternative to the normal document flow

Floated elements are positioned at the left or right edge of their containing element.

To reset floating use the clear property.

See code of menu page for seeing how it can be used.

Relative and Absolute Element Positing

  • Static Positioning – the normal document flow.
  • Relative Positioning – element is positioned relative to their normal position in the document flow. The positioning CSS (offset) properties are: top, bottom, left, right. The element is NOT taken out of normal document flow. If it's offset visiually, its original place is preserved.
  • Absolute positioning – all offsets are relative to the position of the nearest ancestor which has positioning set on it, other than static. By default, html is the only element that has non-static positioning set on it (relative). The element is taken out of the normal document flow

To position elements we start with { position: value; } and then we can write the offsets as { top: value; }
