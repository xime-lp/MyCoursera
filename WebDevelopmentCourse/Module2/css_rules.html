CSS Rules Conflict Resolution and Text Styling

Style Placement

Choosing where to place the styles affects how reusable they are and which style declarations override other ones.

You can put styles in:

  • The head section of the HTML file inside the tag style.
  • Directly on the element in the HTML file: element style="properties"
  • Specifing the styles in a separate css stylesheet and linking it at the head section in the HTML file.

Conflict Resolution

Origin

When we have two declarations in conflict, origin precedence rules kick in as follows: the last declaration wins. Top to bottom within the style and also in the links.

When the declarations are not in conflict, they will merge.

Inheritance

If you specify a property for an element, all the children, grandchildren, etc of the element will inherit those properties.

Specificity

Most specific selector combination wins. The specificity goes like this:

  • Style attribute in an element on the HTML file
  • id tag
  • class, pseudo-class, attribute
  • Element names

We can calculate specificity like a score using these four things.

Styling Text

Specify font family – font-family: 'font', defaults;

Color of text – color : value;

For italic or normal – font-style: value;

For text bold – font-weight: value(word or number);

Size of text – font-size: value;

We can change from lower to upper cas – text-transform: uppercase, lowercase or capitalized;

Position the text – text-align: value;
